# 演讲稿

## 一句话

我们做了一个让 EVE Frontier 经济系统能够自我感知、自我调节的控制台。

---

## 3 分钟版本

大家好，我们的项目叫 Dynamic Economic Regulator。

问题很直接：EVE Frontier 这类开放世界里，经济系统很容易失衡。资源集中在少数区域，价格被局部短缺扭曲，交通拥堵进一步放大失衡。大多数系统只能看到结果，却不能动态调节过程。

我们做了四个链上模块来解决这个问题。

**资源预言机**持续观察每个区域的库存、生产和消耗，判断当前供需状态。

**价格调节器**根据供需自动调整价格，支持 AMM 模式，不需要人工干预。

**金库池**把不同区域的资金放在统一视角下管理，支持跨区域再平衡。

**动态星门**把交通拥堵纳入经济模型，拥堵越高费用越高，反过来影响贸易路线选择。

产品上，我们做了三个界面：Dashboard 展示全局经济状态，Trade 让玩家看到一条路线的真实成本，Admin 提供区域管理和金库监控。

工程上，前端已接入真实 Sui 测试网，合约完整可编译，一条命令完成部署和初始化。

## 5 分钟版本

大家好，我们带来的是 Dynamic Economic Regulator，一个为 EVE Frontier 设计的动态经济调节系统。

先说问题。

开放世界里，经济失衡是常态。某些区域资源堆积，某些区域长期短缺；价格不是因为真实供需变化，而是因为信息不透明被动扭曲；交通路线拥堵，最终玩家体验变成两个字——不公平。

我们的问题是：能不能让经济系统像交通系统一样，被实时观测、实时调节？

**四个链上模块：**

资源预言机采集每个区域的库存、生产率和消耗率，计算供需比例，作为整个系统的数据源。

价格调节器消费供需数据，自动调整价格。它不是手工改表，而是让价格成为系统状态的反映。支持 AMM 流动性池，价格发现更市场化。

金库池管理不同区域的 SUI 余额，支持跨区域转账和自动再平衡。系统能看到哪些区域资金过多、哪些不足，并主动调配。

动态星门把交通拥堵纳入经济模型。拥堵级别 0-5，对应不同费用倍数；支持优先通行证和会员折扣；高峰期附加费、非高峰期折扣，让费用真正反映供需。

**三个产品界面：**

Dashboard 面向所有人，展示今天这个世界发生了什么：价格趋势、资源分布、流向变化、网络状态实时显示。

Trade 面向执行行为。选择起点和目标区域，立刻得到费用估算、预计耗时、拥堵等级，以及优先通行证折扣。点击执行，直接发起链上交易。

Admin 面向治理。查看区域状态、定价配置、金库余额，创建新区域，监控系统告警。

**工程实现：**

前端接入真实 Sui 测试网，合约未部署时自动降级演示数据，不影响现场体验。Trade 页面使用 `useSignAndExecuteTransaction` 执行真实链上交易，成功后显示 Sui Explorer 链接。

Move 合约四个模块完整可编译，部署脚本自动提取对象 ID 并完成初始化，一条命令搞定。

用一句话总结：我们不是在做一个静态经济面板，而是在做一个可以自我调节的文明级经济控制台。

谢谢。

## Demo 讲解顺序

1. 一句话定义问题：资源失衡、价格扭曲、路径拥堵
2. Dashboard：系统"看见了什么"（价格趋势 + 资源分布 + 网络状态）
3. Trade：系统"如何影响一次交易决策"（选区域 → 估算费用 → 执行交易）
4. Admin：系统"如何被治理"（创建区域 + 金库监控）
5. 合约结构 + 部署流程 + 赛后扩展方向
